Documentation

A fast, code-first ORM for SQLite.

BlackHole.Lite gives you Entity Framework-style productivity without a DbContext. Define your entities, call SuperNova once, and the database creates and migrates itself.

.NET 8 SQLite No DbContext Auto migrations LINQ-friendly

Overview

BlackHole.Lite is a lightweight ORM that targets SQLite and is shipped on NuGet as BlackHole.Core.Lite. It is designed to keep the application code clean and the schema in sync with your entity classes — automatically.

  • No DbContext. Configure once at startup with SuperNova and you are done.
  • Schema lives in code. Add a property, restart, the column appears.
  • Familiar API. HasOne, WithMany, HasForeignKey, Include, ThenInclude — just like EF Core.
  • Multi-database. Register many SQLite files and route entities to them by namespace.
  • Raw SQL escape hatch. BHConnection when LINQ is not enough.

Installation

Add the NuGet package to your project:

dotnet add package BlackHole.Core.Lite

Or via the Package Manager Console:

Install-Package BlackHole.Core.Lite
Requirements .NET 8 or higher. SQLite is bundled via SQLitePCLRaw.bundle_e_sqlite3 — no system install required.

Getting Started

Three steps: define an entity, bootstrap the ORM, query it.

1. Define an entity

using BlackHole.Entities;

public class User : BHEntity
{
    public string Name { get; set; } = string.Empty;
    public string Email { get; set; } = string.Empty;
    public int Age { get; set; }
}

Inheriting from BHEntity gives you an auto-generated Id primary key.

2. Bootstrap at startup

using BlackHole.Configuration;

BlackHoleConfiguration.SuperNova(settings =>
{
    settings.AddDatabase("MyAppDb");
});

That single call scans the assembly for entities, creates the SQLite file, and builds tables. If you add a column later, the next run alters the table for you.

3. Use it

using BlackHole.Core;

var users = BHDataProvider.For<User>();

users.InsertEntry(new User { Name = "Mike", Email = "mike@example.com", Age = 32 });

var mike = users.GetEntryAsyncWhere(u => u.Email == "mike@example.com");
var allAdults = users.GetAllEntriesWhere(u => u.Age >= 18);

Defining Entities

Every entity inherits from BHEntity. The base class provides:

  • int Id — auto-increment primary key.
  • int Inactive — internal soft-delete flag (only used if the entity is decorated with [UseActivator]).

Property types map to SQLite types automatically. Supported types include all CLR primitives, DateTime, DateTimeOffset, Guid, byte, uint, ushort, and nullable variants.

public class Order : BHEntity
{
    public int CustomerId { get; set; }
    public DateTime PlacedAt { get; set; }
    public decimal Total { get; set; }
    public string? Notes { get; set; }    // nullable -> nullable column
}

Attributes

Use these attributes on entity properties to fine-tune the generated schema:

AttributeApplies toEffect
[ForeignKey(typeof(T))]PropertyMarks the property as a foreign key referencing entity T.
[Unique]PropertyAdds a UNIQUE constraint on the column.
[NotNullable]Reference propertyForces NOT NULL on a reference type column.
[VarCharSize(n)]String propertyCaps the string column length to n characters.
[UseActivator]ClassEnables soft-delete. Delete* methods set Inactive=1 instead of removing the row.
 
[UseActivator]
public class Customer : BHEntity
{
    [Unique]
    [VarCharSize(120)]
    public string Email { get; set; } = string.Empty;

    [NotNullable]
    [VarCharSize(80)]
    public string DisplayName { get; set; } = string.Empty;
}

Relationships

For richer schemas — foreign keys, delete behaviors, navigation properties — derive from the generic BHEntity<T> and override Configure. The fluent API mirrors EF Core.

public class Order : BHEntity<Order>
{
    public int CustomerId { get; set; }
    public DateTime PlacedAt { get; set; }
    public decimal Total { get; set; }

    public BHIncludeItem<Customer>? Customer { get; set; }

    public override void Configure(RelationBuilder<Order> modelBuilder)
    {
        modelBuilder.HasOne(o => o.Customer)
                    .WithMany(c => c.Orders)
                    .HasForeignKey(o => o.CustomerId)
                    .OnDelete(OnDeleteBehavior.Cascade);
    }
}

public class Customer : BHEntity<Customer>
{
    public string Name { get; set; } = string.Empty;
    public BHIncludeList<Order>? Orders { get; set; }

    public override void Configure(RelationBuilder<Customer> modelBuilder) { }
}

Delete behaviors

ValueMeaning
OnDeleteBehavior.RestrictBlock the delete if any child rows reference it.
OnDeleteBehavior.CascadeDelete child rows along with the parent.
OnDeleteBehavior.SetNullNull out the foreign key on children. Requires the FK column to be nullable.
Heads-up The override is spelled Configure in the current API. This is a known typo and is preserved for back-compat in 0.9.x.

Indexes

Declare indexes inside Configure. Single-column or composite, optionally unique.

public override void Configure(RelationBuilder<Customer> modelBuilder)
{
    // Single column unique index
    modelBuilder.HasIndex(c => c.Email).IsUnique();

    // Composite index
    modelBuilder.HasIndex(c => new { c.LastName, c.FirstName });
}

CRUD Operations

All data access starts from a typed entity context returned by BHDataProvider.For<T>().

var users = BHDataProvider.For<User>();

// Create
int id = users.InsertEntry(new User { Name = "Mike", Age = 32 });
users.InsertEntries(new[] {
    new User { Name = "Anna", Age = 28 },
    new User { Name = "Lila", Age = 41 }
});

// Read
var single = users.GetEntryById(id);
var all    = users.GetAllEntries();

// Update
var u = users.GetEntryById(id);
if (u != null) { u.Age = 33; users.UpdateEntryById(u); }

// Delete
users.DeleteEntryById(id);
users.DeleteAllEntries();

Querying & Filters

Pass LINQ predicates to the *Where overloads. Filters compose naturally and translate to parameterized SQL.

var users = BHDataProvider.For<User>();

var adults    = users.GetAllEntriesWhere(u => u.Age >= 18);
var firstAnna = users.GetEntryWhere(u => u.Name == "Anna");
var hasAdmin  = users.AnyEntry(u => u.IsAdmin);
var howMany   = users.CountEntries(u => u.Age > 30);

// String matching
var ms = users.GetAllEntriesWhere(u => u.Name.StartsWith("M"));

// Composition
var activeAdults = users.GetAllEntriesWhere(u => u.IsActive && u.Age >= 18);

Include & Navigation

Pull related entities in a single query with Include, then go deeper with ThenInclude.

var orders = BHDataProvider.For<Order>()
    .Include(o => o.Customer)
    .GetAllEntries();

foreach (var o in orders)
{
    Console.WriteLine($"{o.Id} - {o.Customer?.Value?.Name}");
}

// Nested
var customers = BHDataProvider.For<Customer>()
    .Include(c => c.Orders)
    .ThenInclude(o => o.Items)
    .GetAllEntries();
Lazy vs Eager BlackHole.Lite is eager-only by design — no surprise N+1 queries. If a navigation is not in Include, it stays null.

Joins

For ad-hoc joins that project into a custom shape, use the join builders on BHDataProvider. Project results into a BlackHoleDto.

public class OrderRow : BlackHoleDto
{
    public int OrderId { get; set; }
    public string CustomerName { get; set; } = string.Empty;
    public decimal Total { get; set; }
}

var rows = BHDataProvider
    .InnerJoin<Order, Customer>()
    .On(o => o.CustomerId, c => c.Id)
    .Where((o, c) => o.Total > 100)
    .Cast<OrderRow>()
    .SelectAll();

Available join types: InnerJoin, LeftJoin, RightJoin, OuterJoin.

Stored Views

If you run the same join again and again, register it once at startup and call it by DTO type.

public class OrderViewSetup : IBHInitialViews
{
    public void DefineViews(BHViewInitializer initializer)
    {
        initializer.CreateView<OrderRow>(
            BHDataProvider.InnerJoin<Order, Customer>()
                .On(o => o.CustomerId, c => c.Id)
        );
    }
}

// Anywhere later
List<OrderRow> rows = BHDataProvider.ExecuteView<OrderRow>();

Transactions

Wrap multiple operations in a BHTransaction. It auto-rolls back if you forget to commit.

using var tx = new BHTransaction();

var customers = BHDataProvider.For<Customer>();
var orders    = BHDataProvider.For<Order>();

int customerId = customers.InsertEntry(new Customer { Name = "Mike" }, tx);
orders.InsertEntry(new Order { CustomerId = customerId, Total = 99.99m }, tx);

tx.Commit();
Tip Every CRUD method has a transactional overload that takes a BHTransaction as the last argument.

Raw SQL

For anything LINQ cannot express, drop down to BHConnection. Parameters keep you safe from injection.

var conn = BHDataProvider.GetConnection();

var args = new BHParameters();
args.Add("minAge", 18);

var adults = conn.Query<User>(
    "SELECT * FROM \"User\" WHERE Age >= @minAge",
    args);

// Single value
int total = conn.QueryFirst<int>("SELECT COUNT(*) FROM \"Order\"", null);

// Non-query
conn.JustExecute("UPDATE \"User\" SET IsActive = 1 WHERE Inactive = 0", null);

Seed Data

Implement IBHInitialData to insert default rows the first time the database is created.

public class DefaultRoles : IBHInitialData
{
    public void SeedData(BHDataInitializer initializer)
    {
        initializer.AddData(new Role { Name = "Admin" });
        initializer.AddData(new Role { Name = "User" });
    }
}

The seeder runs automatically once, on first DB creation. Subsequent runs do nothing.

Multiple Databases

Register more than one database and route entities to them by namespace.

BlackHoleConfiguration.SuperNova(settings =>
{
    settings.AddDatabase("UsersDb",    "MyApp.Domain.Users");
    settings.AddDatabase("ProductsDb", "MyApp.Domain.Products");
});

// Address a specific DB at query time
var users = BHDataProvider.For<User>("UsersDb");
var skus  = BHDataProvider.For<Sku>("ProductsDb");

Soft Delete

Decorate any entity class with [UseActivator] and Delete* calls flip Inactive = 1 instead of removing the row. GetAll* calls automatically hide inactive rows.

[UseActivator]
public class Account : BHEntity
{
    public string Email { get; set; } = string.Empty;
}

var accounts = BHDataProvider.For<Account>();
accounts.DeleteEntryById(42);   // sets Inactive = 1, row stays
var live = accounts.GetAllEntries(); // id=42 not returned
Permanent delete Drop down to BHConnection.JustExecute with a raw DELETE if you really need to remove the row.
